internetwork      protocol                           IP 		 网际协议

Internet   group        management  protocol         IGMP        internet组管理协议


运输层:运输层主要为两台主机上的应用程序提供端到端的通信，协议有:
transfer   control      protocol                     TCP         传输控制协议               T C P为两台主机提供高可靠性的数据通信。它所做的工作包括把应用程序交给它的数据分
																							成合适的小块交给下面的网络层，确认接收到的分组，设置发送最后确认分组的超时时钟
user       datagram     protocol                     UDP         用户数据报协议				把称作数据报的分组从一台主机发送到另一台主机，但并不保证该数据报能到达另一端。任何必需的可靠
																							性必须由应用层来提供
区别:TCP传输可靠的数据给网络层；T C P服务器是并发的，而 U D P服务器是重复的；

IP路由器：IP Router
网关（gateway）现在网关这个术语只用来表示应用层网关：一个连接两种不同协议族的进程（例如，TCP/IP和IBM的SNA），它为某个特定的应用程序服务（常常是电子邮件或文件传输）。
互联网络信息中心（ Internet Network Information Centre），称作InterNIC。InterNIC只分配网络号。主机号的分配由系统管理员来负责
现在， N I C只负责处理国防数据网的注册请求
网桥：连接两个网络，只适用于数字作为唯一标识网络的节点。
交换机：连接两个或两个以上的网络
路由器：连接多个不同类型的网络
有三类I P地址：单播地址（目的为单个主机） 、广播地址（目的端为给定网络上的所有主机）以及多播地址（目的端为同一组内的所有主机）。
两种应用编程接口（API）：T L I（运输层接口）、 XTI（X/Open运输层接口），实际上是TLI的一个超集。、
4个字节的32 bit值以下面的次序传输：首先是 0～7 bit，其次8～15 bit，然后1 6～23 bit，最后是24~31 bit。这种传输次序称作 big endian字节序,又称作网络字节序
目前的协议版本号是 4，因此I P有时也称作I P v 4
以太网的最小帧长为 4 6字节。主机也要求不能接收超过 576字节的数据报，UDP的应用都限制用户数据报长度为 512字节，小于576字节。
IP层在内存中有一个路由表，表中的每一项都包含下面这些信息：目的I P地址、下一站（或下一跳）路由器的IP地址、标志、为数据报的传输指定一个网络接口。
